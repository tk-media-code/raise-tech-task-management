# タスク管理アプリ プロトタイプ（動作確認用モック）

[要件定義書](../docs/requirements.md) と [画面構成・画面遷移](../docs/requirements/03-screens.md) をもとに、アプリの動作イメージを確認するための**モック**です。本番実装ではありません。

## 開き方

サーバーは不要です。`index.html` をブラウザで開くだけで動作します。

- `prototype/index.html` をダブルクリックする
- または、ブラウザのウィンドウに `index.html` をドラッグ＆ドロップする

Chrome / Edge / Firefox / Safari の最新版で動作確認しています。

## 技術構成

- HTML / CSS / JavaScript のみ（フレームワーク・ビルドツール・npm依存なし）
- `index.html` … 画面の器（ヘッダー・メイン領域・各モーダル）
- `styles.css` … 全スタイル（レスポンシブ対応含む）
- `app.js` … 状態管理・画面描画・イベント処理
- データは **localStorage** に保存されます（キー: `taskmgmt.prototype.v1`）。ブラウザを閉じても、リロードしてもデータは保持されます。
- 初回アクセス時は要件定義書のワイヤーフレームを再現したサンプルデータ（ボード「仕事」「家事」等）が自動的に投入されます。
- ヘッダー右端の「↺」ボタンで、いつでもサンプルデータへリセットできます（現在のデータはすべて消えます）。

## モック上の割り切り（本番実装時に要検討）

実装にあたり、要件定義書には明記されていない、または意図的に簡略化した点です。

- **タッチ端末でのドラッグ＆ドロップ**：PC（マウス）でのドラッグ＆ドロップは HTML5 Drag and Drop API で実装していますが、iOS Safari 等ではこのAPIのタッチ対応が不安定なため、タッチ端末では各カードの「⋯」メニュー（スマホ表示では「移動▾」ボタン）からのステータス変更を確実な代替手段としています（要件5.3で定義された代替手段そのものです）。
- **横断ビューでの別ボードへのドラッグ**：横断ビュー内でカードを別ボードのセクションへドラッグしても移動しない仕様にしています（カードを別ボードへ付け替える機能は要件のスコープ外のため）。同一ボード内での列間移動（ステータス変更）は横断ビューでも可能です。
- **アーカイブできるのは「完了」ステータスのカードのみ**：要件5.7が「完了したカードのアーカイブ」を前提としているための制約です。未着手・作業中のカードをアーカイブしたい場合は、先にステータスを「完了」に変更してください。
- **検索のラベル絞り込みは AND 条件**：複数ラベルを選択した場合、選択した**すべて**のラベルが付いているカードのみを表示します（OR条件ではありません）。
- **カードのタイトル編集**：受け入れ条件には明記されていませんが、カード詳細モーダルの見出し（タイトル表示部分）をそのまま編集可能にしています（CRUDのUpdateとして自然なため）。
- **ベーシック認証・HTTPS（8.2）**：ホスティング層で完結する要件であり、アプリ本体（本モック）には影響しないため未実装です。
- **新規カードの追加列**：「＋カードを追加」は仕様上の初期値に合わせ、「未着手」列のみに設置しています。

## データのリセット・削除

- ヘッダーの「↺」ボタン：確認の上、サンプルデータへ完全リセットします。
- ブラウザの開発者ツールで `localStorage.removeItem('taskmgmt.prototype.v1')` を実行してリロードしても、同様に初期状態に戻ります。
